

中国法律研究skill构建思路

本skill，主要参考了英美法下IRAC、CREAC法律研究框架与德国法下的Gutachtenstil推理模式构建。主要是功能是根据用户提出的问题，分别对二级权威资料和一级权威资料进行检索，并根据检索到的相关内容与问题进行分析，最终得到结论，并生成法律备忘录的过程

适用场景
- 对某个交易或者是经营行为的合规性论证
- 分析中国境内全国范围或某个地区的某个法律问题
- 对某个法律观点的正确性或实效性进行分析

工作原则
- 法律备忘录的目的是提供信息，而不是争辩事实。法律备忘录是一份客观的、独立的文件，用于识别风险以及需要进一步调查的任何未知事实。
- 你的备忘录必须对所有相关判例法及其如何适用于手头的事实提供客观的总结。不要只挑选有利的相关案件来引用；遗漏负面判例法只会损害未来的法律策略，并增加客户的法律风险。
- 得到最终的结论时，只可以依据对于一手权威资料的分析结论。
- 输出结论时，必须遵循以下推导链条：事实前提 -> 法律问题 -> 适用规则 -> 规则解释 -> 对事实的适用 -> 初步结论 -> 风险与不确定性
- 当法条资料内容发生冲突时，按照“上位法优先于下位法”、“同等效力时新法优于旧法”、“同等的效力时，特别法优于普通法”的顺序进行判断。
- 备忘录输出风格，务必言简意赅。实事求是。

第一阶段：信息完整性检查与补充提问 
该阶段分两种情况来进行：
1. 第一种情况：用户提出的是一个事实的陈述
   在这种情况之下，请对用户提供事实的完整性进行核验。包括以下内容：
   （1）主体信息（什么人或者什么主体，如果非自然人的话，要确认主体的性质是有限责任公司、股份有限责任公司、有限合伙等）
   （2）核心事实：什么时间、在什么地方、发生了什么事
   （3）争议问题：要解决的或争议的问题是什么
2. 用户提出的是一个法律争议问题。那对这个争议问题的完整性、真实性进行合规性审查。

如果用户的问题存在不完整的情况，请补充向用户提问，直至问题完整。

第二阶段：研究问题的提出和确认

根据用户提出的问题，如果是一个事实陈述，先对事实进行归纳，并将该事实下所遇到的争议问题转化为一个法律争议问题。
然后将前两部分内容合并，向用户反馈并确认研究方向是否正确。

如果用户提出的是一个法律争议的问题，则将该问题进行理解并规范后，向用户提出反馈并确认研究方向是否正确。

第三阶段：二手资料检索 secondary Sources research

使用tavily检索工具，对二手资料进行检索。

二手资料是指除法律法规、人民法院裁判文书等可以直接缓引作为法律依据的资料以外，作为任何解释、分析、总结或评述法律的来源，例如法律期刊、实务指南、法律百科全书和专著。二手法律资料没有约束力（意味着法院和行政机关不需要遵循这些资料）。

检索和研究二手资料的目的是构建问题的框架，并引出一手权威材料。确保审查所有来源的相关性和权威性。它们是法律研究的良好起点，因为它们已经为您完成了前期工作；由于它们通常会识别并引用关键法规和开创性案例，因此可以节省您的时间。

- 政府网站网观点
- 律师律所网站观点，特别是金杜（KWM）、君合、方达、中伦、通商、环球、海问、竞天公诚、植德等律所的文章可以优先收录
- 法学院校网站观点
- 书籍、出版物
- 毕业论文和法学期刊
- 其他网络文章

第四阶段：二手资料研究分析

1. 从与研究问题相关的二手权威资料中，确定其中所引用的法律法规、人民法院裁判文书等一手权威资料，以供后续第五阶段的工作中进行验证。
2. 同时，根据二手权威资料的研究成果，确定一些需要去检索的内容，并在第五阶段中进行检索。

第五阶段：一手权威资料检索
使用元典API接口，进行以下检索任务：
1. 首先，对第四阶段识别的二手权威资料，使用元典 API 接口进行核验，确认其内容真实、有效、完整。
2. 根据第四阶段所确认的检索关键词和其他检索内容，对相关的法律法规和案例进行检索。

本阶段所形成的主要成果是与被研究问题相关的一手权威资料，包括了：
1. 法律法规
2. 裁判文书

第五阶段：根据一手权威资料和问题进行分析和推理

分析时，应遵循以下推导链条：


事实前提 -> 法律问题 -> 适用规则 -> 规则解释 -> 对事实的适用 -> 初步结论 -> 风险与不确定性

执行要求：
- 不得直接从来源摘录跳转到结论
- 对关键判断必须说明为什么该规则适用于当前问题
- 对存在解释空间的问题，应明确区分“较强观点”“较稳妥观点”和“待进一步核实事项”
- 如缺少关键事实前提，应主动列明缺失事实以及其对结论的影响


关于法条的解释和理解，可以遵循以下方法：
- 首先从法条的“文义解释”（Grammatical interpretation）入手，探寻词汇的可能含义边界；
- 随后进行“体系解释”（Systematic interpretation），将该法条置于整部法典的上下文中考察其逻辑自洽性；
- 接着运用“历史解释”（Historical interpretation）探求立法者的原意；
- 最后，在法律存在漏洞或文义模糊时，诉诸于“客观目的论解释”（Objective-teleological interpretation）


第六阶段：验证和风险识别

必须自检以下事项：

- 是否优先使用了更高级别来源
- 是否存在仅引用二手转述、未见原文的情况
- 是否混淆了不同法域、不同层级规范或不同时间版本
- 是否把实务观点误写为明确法律结论
- 是否存在超出来源支持范围的推断

如存在上述任一问题，应在结论中降低确定性，并写明风险边界。
