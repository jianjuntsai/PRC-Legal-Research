## 一、项目定位

### 核心定位

构建一个 Claude Code Skill（`/法律研究`），实现系统性中国法律研究：通过 **Tavily 检索二手资料**构建框架 → 通过**元典 API 获取一手权威资料**验证和补充 → 输出**法律研究备忘录**。

### 适用场景

- 对某个交易或经营行为的合规性论证
- 分析某一国家或地区的某个法律问题
- 对某个法律观点的正确性或时效性进行分析

### 与参考项目的对比

| 维度 | Golden2002 / Vibelawyer | 本项目 |
|------|------------------------|--------|
| 数据来源 | 依赖 AI 训练知识 + 威科先行 API | **元典 API**（法规/法条/案例）+ **Tavily**（二手资料网络检索） |
| 法条内容 | AI 记忆中的法条（可能不准确） | **API 实时返回法条原文**，确保准确性和时效性 |
| 案例来源 | AI 记忆或威科 API | **权威案例 + 普通案例**，均来自元典 API |
| 核心优势 | 流程和格式设计详尽 | **一手资料由 API 验证，二手资料由网络检索，两者交叉印证** |

### 设计决策

- **适用范围**：第一版只做法律研究，预留扩展接口（后续可加企业合规）
- **案例深度**：默认用摘要，判断需要深入分析时再调 `get_case_detail` 获取全文
- **检索语言**：按问题自动判断，国内问题中文为主，涉外问题加入英文

### 与参考 Skill 的取舍

**采纳的设计**（来自 Golden2002 / Vibelawyer）：
- ✅ 信息完整性检查和补充提问流程
- ✅ 事实确认后才进入检索（强制流程）
- ✅ 法条引用必须包含完整要素（名称/文号/生效时间/原文/效力状态）
- ✅ 条文互引梳理
- ✅ 法规冲突适用分析
- ✅ 时效与期限梳理

**不采纳的设计**：
- ❌ 用户身份识别与差异化服务 — 统一按律师/法务水准输出
- ❌ 威科先行 API 集成 — 已有元典 API
- ❌ Word 文档输出 — 先专注 Markdown
- ❌ 常见法律领域速查表 — 改用 API 实时检索

**本 Skill 独有的设计**：
- ⭐ 二手→一手的两阶段检索：Tavily 找线索 → 元典 API 验证和扩展
- ⭐ 信息来源强制标注：`[元典API]` / `[Tavily]` / `[AI分析]`
- ⭐ 法律解释方法链：文义→体系→历史→目的论
- ⭐ 交叉印证思维：法条 + 案例 + 既有研究三方验证

## 三、文件 1：`tavily_search.py`

Tavily 检索封装，提供三个函数用于二手法律资料检索。

```python
"""
Tavily 检索封装 — 用于法律研究中的二手资料检索
"""

import json
import sys
import os

sys.path.insert(0, os.path.dirname(__file__))
from config import TAVILY_API_KEY
from tavily import TavilyClient

client = TavilyClient(api_key=TAVILY_API_KEY)


def search_secondary_sources(query, max_results=10, search_depth="advanced",
                             include_domains=None, exclude_domains=None):
    """
    检索二手法律资料
    - query: 检索关键词
    - max_results: 返回结果数，默认10
    - search_depth: "basic" 或 "advanced"（更深入），默认 advanced
    - include_domains: 限定域名列表（如律所网站）
    - exclude_domains: 排除域名列表
    返回: list[dict]，每条含 title/url/content/score
    """
    kwargs = {
        "query": query,
        "max_results": max_results,
        "search_depth": search_depth,
    }
    if include_domains:
        kwargs["include_domains"] = include_domains
    if exclude_domains:
        kwargs["exclude_domains"] = exclude_domains

    result = client.search(**kwargs)
    return result.get("results", [])


def search_lawfirm_articles(query, max_results=5):
    """
    专门检索头部律所的法律分析文章
    优先：金杜、君合、方达、中伦、通商、环球、海问、竞天公诚、植德、汉坤
    """
    lawfirm_domains = [
        "kwm.com",           # 金杜
        "junhe.com",         # 君合
        "fangda-partners.com",  # 方达
        "zhonglun.com",      # 中伦
        "tongshang.com",     # 通商
        "haiwen-law.com",    # 海问
        "hankunlaw.com",     # 汉坤
        "jingtian.com",      # 竞天公诚
        "meritsandtree.com", # 植德
        "globe-law.com",     # 环球
    ]
    return search_secondary_sources(
        query=query,
        max_results=max_results,
        include_domains=lawfirm_domains,
    )


def search_government_interpretations(query, max_results=5):
    """
    检索政府网站的政策解读、答记者问等
    """
    gov_domains = [
        "gov.cn",
        "npc.gov.cn",
        "court.gov.cn",
        "spp.gov.cn",
        "moj.gov.cn",
    ]
    return search_secondary_sources(
        query=query,
        max_results=max_results,
        include_domains=gov_domains,
    )


# ──────────────────────────────────────────────
# 快速测试
# ──────────────────────────────────────────────

if __name__ == "__main__":
    keyword = sys.argv[1] if len(sys.argv) > 1 else "合同违约责任 法律分析"

    print(f"=== 综合检索：{keyword} ===")
    results = search_secondary_sources(keyword, max_results=3)
    for r in results:
        print(f"- [{r.get('score', 0):.2f}] {r['title']}")
        print(f"  {r['url']}")
        print(f"  {r.get('content', '')[:100]}...")
        print()

    print(f"\n=== 律所文章检索：{keyword} ===")
    results = search_lawfirm_articles(keyword, max_results=3)
    for r in results:
        print(f"- {r['title']}")
        print(f"  {r['url']}")
        print()
```

## 工作原则

1. **客观性**：法律备忘录是客观文件，用于识别风险，不争辩事实。不要只挑选有利的案件引用，遗漏负面判例法只会损害法律策略。
2. **一手资料为王**：最终结论只能依据一手权威资料（法律法规、裁判文书）。二手资料（律所文章、学术论文）只用于构建框架和引出线索。
3. **信息来源透明**：每条引用必须标注来源：
   - `[元典API]` — 来自元典 API 的法条/案例原文
   - `[Tavily]` — 来自 Tavily 检索的二手资料
   - `[AI分析]` — 你自己的推理和分析（必须明确标注）
4. **时效性敏感**：关注法规的 `sxx`（时效性）字段，优先引用"现行有效"的法规。对已失效或已修改的法规必须明确标注。
5. **法规冲突处理**：上位法优于下位法；同等效力时新法优于旧法；同等效力时特别法优于普通法。
6. **言简意赅，实事求是**。

## 七阶段工作流

### 第一阶段：信息完整性检查与补充提问

分两种情况：
1. **事实陈述型**：检查主体信息（什么人/主体，主体性质）、核心事实（时间/地点/事件）、争议问题（要解决什么）
2. **法律问题型**：检查问题的完整性和真实性

如果信息不完整，向用户提问补充。不要在信息不完整时就开始研究。

### 第二阶段：研究问题的提出和确认

1. 如为事实陈述 → 归纳事实，将争议转化为法律问题
2. 如为法律问题 → 规范化表述
3. 向用户反馈并确认研究方向是否正确

### 第三阶段：二手资料检索

使用 Tavily 检索工具，目的是**构建问题框架，引出一手资料线索**。

检索策略：
1. 调用 `search_lawfirm_articles(query)` — 检索头部律所文章
2. 调用 `search_government_interpretations(query)` — 检索政府解读
3. 调用 `search_secondary_sources(query)` — 综合检索

语言策略：国内法律问题用中文检索；涉外问题同时用中英文检索。

关注要点：
- 律所文章中引用了哪些法律法规（法规名 + 法条号）
- 文章中提到了哪些案例（案号或案件名）
- 政府的政策解读和答记者问

### 第四阶段：二手资料分析

从第三阶段检索结果中提取：
1. **引用的法律法规清单** — 法规名称 + 法条号（待验证）
2. **引用的案例** — 案号或案件名（待核实）
3. **关键词线索** — 用于第五阶段元典 API 检索的扩展关键词

### 第五阶段：一手权威资料检索与验证

**5.1 验证二手资料引用**

对第四阶段提取的每条法规/法条，调用元典 API 验证：
```python
get_fatiao_detail(fgmc="法规名称", ftnum="法条号")
```
确认法条是否真实存在、内容是否与二手资料描述一致、是否现行有效。

**5.2 扩展检索**

基于研究问题和第四阶段线索：
```python
search_fatiao(keyword="关键词", sxx="现行有效", top_k=15)
search_fagui(keyword="关键词", sxx="现行有效", top_k=5)
search_qwal(qw="关键词", ajlb="案件类别", top_k=10)
search_ptal(qw="关键词", ajlb="案件类别", top_k=10)
```

**5.3 按需获取详情**

默认使用检索结果中的摘要/llm_content。当以下情况时获取全文：
- 案例与研究问题高度相关，需要分析裁判说理
- 法规需要查看完整条文结构
```python
get_case_detail(type="qwal", id="案例ID")
get_fagui_detail(fgmc="法规名称")
```

### 第六阶段：分析与推理

基于一手权威资料进行法律分析：

**法律解释方法链**（按顺序使用）：
1. **文义解释** — 法条文字的可能含义边界
2. **体系解释** — 法条在整部法律中的上下文逻辑
3. **历史解释** — 立法者原意（参考立法释义、答记者问）
4. **客观目的论解释** — 法律漏洞或文义模糊时

**推导链条**：
事实前提 → 法律问题 → 适用规则 → 规则解释 → 对事实的适用 → 初步结论 → 风险与不确定性
